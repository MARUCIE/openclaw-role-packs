# W1 · 交付 Checklist（pre / on-site / post 三段）

> 配合 [Wave Digest](../../doc/wave-digest.md) 的 HITL 检查点
> cohort + facilitator 各自在三个时点对照此清单自查

## 阶段 1 · Pre-workshop（T-3 ~ T-1，2026-05-18 周一 ~ 2026-05-20 周三）

### Pre-1 · 工具空投到位（facilitator 责任）

- [ ] T-3 (周一)：本目录全套素材已发送到 cohort 群（含 README + 5 件工具套件 + 3 个深度版 + checklist）
- [ ] T-3 (周一)：cohort 收到工具包后已确认接收（群内回执 ≥ 5 人确认收到）
- [ ] T-3 (周一)：CLAUDE.md 配置文件已在 cohort 内 ≥ 3 个项目中冒烟（验证可加载）

### Pre-2 · cohort 自学（cohort 责任）

- [ ] T-2 (周二)：cohort 成员通读完 README + baseline + INDEX（≥ 30 min）
- [ ] T-2 (周二)：cohort 已选定本周要做的 1 个真实任务（不要工作坊现场拍脑袋）
- [ ] T-1 (周三)：cohort 已 dry-run 至少 1 条 prompt（族 1 任一 D 类）并把结果发到群

### Pre-3 · 物料准备（facilitator + the program owner）

- [ ] T-1 (周三)：SOP 流程图打印 A2 海报上墙（workshop 现场）
- [ ] T-1 (周三)：评审录音设备 / 屏幕共享通道已测试
- [ ] T-1 (周三)：facilitator + co-facilitator 已对齐分工（谁讲族 1 / 族 2 / 族 3）

## 阶段 2 · On-site workshop（2026-05-21 周四）

### On-1 · 开场（30 min）

- [ ] cohort 自我介绍 + 各自 dry-run 反馈分享（每人 ≤ 2 min）
- [ ] facilitator 讲 baseline 表的"为什么这是北极星"
- [ ] 现场敲定本周作业的 3 个 pod（每 pod 2-3 人 + 1 个真实任务）

### On-2 · 主体（5 h，3 pod 并行）

- [ ] 每个 pod 选 1 个 Day 1-5 节点开始（不要从头到尾全跑，挑最痛的）
- [ ] facilitator 巡回 ≤ 30 min/pod 干预（介入太多会让 cohort 不学）
- [ ] 每个 pod 中场 (~ 2.5 h) 做一次 5 min 站会同步进度
- [ ] 收尾前每 pod 出 1 张"输出截图"（任何形态）贴 wall

### On-3 · 收尾（30 min）

- [ ] 3 个 pod 各 5 min 展示输出 + 1 个最大教训
- [ ] facilitator 用 5 min 收束："本周 Skill / SOP / 文档模板的反馈是什么"
- [ ] 现场记录"工具反馈"段（见本文件最末）
- [ ] 现场敲定 T+1 即时回顾时间（30 min 锁定 2026-05-22 周五的具体时段）

## 阶段 3 · Post-workshop（T+1 ~ T+8）

### Post-1 · T+1 即时回顾（2026-05-22 周五 · 30 min）

- [ ] cohort liaison + the program owner 30 min 视频会议（必录）
- [ ] 回写 [wave-digest.md](../../doc/wave-digest.md) W1 段的 Worked / Broke and patched / One advice
- [ ] 录音链接存到 wave-digest.md W1 段的 Recording 字段
- [ ] W2 cohort liaison 提名（_[待 D2 命名]_ 此时填名）

### Post-2 · T+8 中期回顾（2026-05-30 周五）

- [ ] cohort 自报本周 baseline 任务实际节省时间 vs 表中 After 列
- [ ] 任一行实际值偏离 After ≥ 30% → 标记"待调整"，纳入 Buffer 周（W5）修订
- [ ] cohort 自建 ≥ 1 个新 Skill 的截图证据
- [ ] 30 天采纳追踪 Owner 已分配（cohort 内自荐）

### Post-3 · 30 天 Capstone 输入准备（2026-06-19 周四）

- [ ] W1 cohort 准备 30 天采纳数据 + 1 个跨部门复用样板
- [ ] 复用样板已得到目标部门预初步确认（不是单方面"我觉得他们能用"）

## Quality Threshold

如果本 checklist 完成后**所有项都是"OK 通过"**且**工具反馈段全部留白** —— 这是 Wave Digest 契约里"all worked, nothing broke"的反模式，**视为零反馈，触发 AI 专家追问**。

每一波都必然有破绽 / 摩擦 / 临时绕过。把它们写出来 cohort 才能真的学到东西。
