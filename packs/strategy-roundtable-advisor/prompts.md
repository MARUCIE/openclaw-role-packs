# 战略圆桌顾问 · Prompt Library

## Prompt 01 · Core Question Intake

用于用户只给了一个模糊想法时，把它压缩成可分析问题。

```text
先复述我真正要决策的问题。
列出缺失背景，只问最关键的 1 个问题。
如果已有足够信息，直接进入分析，不要继续追问。
输出：核心问题、背景字段、下一步分析入口。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 02 · Cognitive Skeleton Routing

用于先选模型，不急着展开圆桌。

```text
把当前问题匹配到 cognitive-skeleton 的 9 类场景。
选择 2 个 Decision Framework lens 和 2 个 PM framework。
说明组合顺序：先理解为什么，再决定怎么做。
输出：3-5 个模型、适用理由、使用顺序。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 03 · Expert Roundtable Report

用于直接启动多专家圆桌。

```text

使用 8 位默认专家独立分析。
进行两轮交叉质询：冲突发现、争议收敛。
输出结论先行的金字塔报告。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 04 · Strategy Option Comparison

用于 A/B/C 战略选项比较。

```text
列出每个选项的适用条件。
按收益、风险、资源、时间窗口、可逆性评分。
指出看似保守但实际风险更高的选项。
输出推荐选项、备选路径、触发条件。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 07 · Risk Red Team

用于方案看起来太顺时做压力测试。

```text
列出会让方案失败的 5 个前提。
为每个前提设计早期预警信号。
给出预防措施和应急动作。
输出 go / no-go / pivot 条件。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 08 · Reflection Promotion

用于分析完成后的知识沉淀。

```text
只提取可复用规则、决策、反模式。
拒绝沉淀原始讨论和一次性观察。
写出证据、适用范围、失败模式。
输出 cognitive-reflection 候选条目。
```

使用边界：
- 需要核心问题。
- 需要相关背景。
- 输出必须包含行动路径。
- 输出必须包含失败信号。

## Prompt 09 · Full Workshop Mode

把前面 8 个 prompt 串成一次完整战略工作坊。

```text
先做 intake，再做 cognitive-skeleton 路由。
对明确的战略问题启动多专家圆桌。
对关键争议做 red team。
按业务或产品方向承接下游管线。
最后只沉淀可复用规则。
```
