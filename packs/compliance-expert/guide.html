ft-business-analyst
skills/compliance/ft-business-analyst/SKILL.md

Use when: understanding business scenarios, identifying tax-related

是什么

业务分析师是财税合规链路的第一道闸口（首环判定），把客户口语化的"我们做什么生意"翻译成结构化的涉税事件清单，让后续的会计师、税务师、合规审计师拿到的不是模糊描述而是可申报、可稽查、可追溯的业务事实，直接降低税种鉴定错配（错误适用增值税税率）和经营范围漏报触发的稽查风险。

怎么用

  1. 先问纳税人身份（小规模纳税人 / 一般纳税人 / 小型微利企业），定下后续所有税率与优惠政策的判定基线，避免一开始就把简易计税误用为一般计税。
  2. 按经济业务六大分类（销售 / 采购 / 投资 / 融资 / 薪酬 / 资产）逐项追问交易实质，把"卖东西"拆解到货物销售（13% 增值税）还是服务（6% 增值税）这一颗粒度。
  3. 对照行业特征速查表标记常见坑（如制造业的暂估入账长期不回票、贸易行业的虚开风险），形成行业专属的合规预警清单。
  4. 识别关联交易（持股 25% 以上 / 实际控制 / 亲属关系），在事前留出独立交易原则定价空间和同期资料文档准备时间，避免事后被税务机关核定调整。
  5. 把结构化的业务事件清单（含税种判定、时点确认、金额口径）输出给会计师和税务师，并在客户经营范围变更或新增业务线时同步更新一次。

    F --> G[交付会计师/税务师]
ft-compliance-auditor
skills/compliance/ft-compliance-auditor/SKILL.md

Use when: risk identification, compliance checking, Golden Tax IV alerts,

是什么

合规审计师是财税处理的最后一道防线（终审闸口），在会计师做完账、税务师算完税之后做全面交叉核验，确保业务流、合同流、发票流、资金流四者一致，专门拦截金税四期（多数据源比对系统）触发预警的高风险事项，把稽查触发概率与补税罚款金额降到最低。

怎么用

  1. 接到月度账务之后，先对照 TOP 20 合规风险清单逐项排查虚开发票、隐匿收入、私户收款（个人账户大额交易报告标准：单笔或当日累计 5 万元现金 / 20 万元转账）。
  2. 在每月质检期（16-24 日）执行 10 项月度质检清单（银行余额调节 / 往来账龄 / 进销项匹配 / 工资与个税人数一致性等），把问题拦在萌芽阶段。
  3. 比对金税四期五大数据源（银行流水 / 增值税发票 / 企业所得税 / 社保人数 / 个税人数），偏差超过 20% 的指标立即定位还原并准备情况说明。
  4. 按风险等级矩阵（低 / 中 / 高 / 极高）输出整改优先级和应对动作，极高风险（涉及虚开、隐匿收入、骗税）立即停止违规行为并升级至专业税务律师。
  5. 在汇算清缴前做一次全年纳税调整事项专项质检（12 类常见调整项），避免年底集中爆雷导致整改成本翻倍。

flowchart LR
    A[月度账务] --> B[TOP 20 风险扫描]
    B --> C[10 项质检清单]
    C --> D[金税四期数据比对]
    D --> E[风险等级评定]
    E --> F[整改方案 / 应对材料]
ft-compliance-checklist
skills/compliance/ft-compliance-checklist/SKILL.md

Use when: generating compliance checklists, monthly/quarterly/annual

是什么

合规检查清单生成器按行业、企业规模、纳税人类型自动产出结构化的月度 / 季度 / 年度合规检查清单（含征期日历、行业特殊事项、地方征期差异），让代账团队与企业自查环节有可执行的"该查什么"路线图，避免漏申报、漏检查触发税务机关异常户管理（限制发票领用、纳税信用降级）。

怎么用

  1. 先确认客户三个关键参数（行业代码 / 纳税人类型 / 企业规模），决定基础清单走小规模季报、一般纳税人月报，还是叠加小型微利企业优惠条款。
  2. 按代账 41 任务模型时间窗口（1-5 日采集 / 6-15 日征期 / 16-24 日质检 / 25-31 日准备）生成月度清单，把"什么时候做什么"对齐到征期日历。
  3. 叠加行业特殊项（建筑业项目预缴 / 房地产土地增值税清算 / 高新技术企业研发费用辅助账 / 出口退税申报），覆盖通用清单无法覆盖的行业风险。
  4. 季度末追加企业所得税预缴、财务报表报送、税负率自查项；年度末追加汇算清缴、工商年报、社保基数调整、税收优惠备案等一次性事项。
  5. 汇算清缴季提前 3 个月（10 月起）启动准备清单，把资产盘点、坏账确认、关联交易整理、研发费用归集复核分摊到 Q4 而非压到次年 1 月。

架构图

flowchart LR
    A[行业+规模+纳税人类型] --> B[基础清单选型]
    B --> C[时间窗口分配]
    C --> D[行业特殊项叠加]
    D --> E[征期日历对齐]
    E --> F[交付 ft-compliance-auditor]
ft-internal-audit
skills/compliance/ft-internal-audit/SKILL.md

Use when: user needs internal control evaluation, process audit, fraud

是什么

内部审计专家基于 COSO（内部控制框架）对企业关键业务流程的控制有效性做独立评价，识别授权 / 执行 / 记录 / 保管四职能分离失效、舞弊红旗信号（管理层凌驾内控、异常关联交易、年末大额调整分录）和 IT 系统自动化控制缺陷，输出可追溯的审计工作底稿和分级整改建议，直接降低重大缺陷被监管追责和上市公司披露违规的概率。

怎么用

  1. 按 COSO 五要素（控制环境 / 风险评估 / 控制活动 / 信息与沟通 / 监控）对关键业务流程做穿行测试，先理清"制度上写了什么"和"实际做了什么"之间的差距。
  2. 验证四职能分离（授权 / 执行 / 记录 / 保管不能集中在同一人），重点检查采购、销售、资金、薪酬四条主线的关键控制点是否被绕过。
  3. 按交易性质选择审计抽样方法（统计抽样推断总体 / 判断抽样针对异质交易 / 金额超重要性水平的全查），避免对判断抽样结果做"总体没问题"的过度推断。
  4. 把审计发现按缺陷严重程度分级（一般缺陷 / 重要缺陷 / 重大缺陷），重大缺陷立即报告审计委员会并评估上市公司对外披露义务。
  5. 跟踪整改不以"整改完成率"为唯一指标，重点监控同类问题的重复发生率，防止形式主义整改。

架构图

flowchart LR
    A[COSO 五要素评估] --> B[穿行测试]
    B --> C[控制测试]
    C --> D[实质性测试]
    D --> E[缺陷分级 / 整改建议]
    E --> F[审计底稿归档]
ft-risk-assessment
skills/compliance/ft-risk-assessment/SKILL.md

Use when: evaluating tax compliance risk, benchmarking financial indicators

是什么

风险评估矩阵是企业税务合规风险的预警引擎，按收入 / 成本 / 费用 / 资产 / 往来五维度对企业财务指标做行业基准对标和异常检测，量化输出低 / 中 / 高 / 极高四级风险评分（0-100 分），在税务机关发现问题之前生成可执行的整改路线，把稽查触发概率和潜在补税滞纳金金额双重压低。

怎么用

  1. 先采集近 3 年财务报表、纳税申报表、发票数据、银行流水做数据基线，避免只看一年导致的趋势误判。
  2. 按国民经济行业代码 2 位级别确定行业分类，调取该行业的毛利率、增值税税负率、企业所得税税负率参考区间做横向对标。
  3. 跑高危指标清单（连续 3 个月零申报但银行有流水 / 进项销项长期倒挂 / 税负率偏离行业均值超 50% / 存货账实差异超 10%）和中危指标清单（毛利率同比波动超 10 个百分点 / 关联方交易占比超 50%）做异常筛查。
  4. 区分趋势异常（毛利率从 30% 突降到 15%）与绝对值异常（毛利率持续低于行业），趋势异常的预警价值通常更高。
  5. 输出预警报告时每项异常必须给出三件套（合理原因+所需证据 / 整改措施+时间窗口 / 不整改后果+罚款估算），而不是只列问题不给方案。

架构图

flowchart LR
    A[3 年财务数据] --> B[行业基准对标]
    B --> C[五维度异常检测]
    C --> D[风险评分 0-100]
    D --> E[整改优先级]
    E --> F[预警报告输出]
ft-tax-advisor
skills/compliance/ft-tax-advisor/SKILL.md

Use when: tax type determination, tax rate application, tax filing,

是什么

税务师把会计师产出的账务数据翻译成 18 个税种（增值税 / 企业所得税 / 个人所得税 / 附加税 / 印花税 / 房产税 等）的合规申报结果，负责税种鉴定、税率适用、政策解读、优惠应享尽享，直接决定企业是依法纳税还是因为税率错配（混合销售与兼营判定错误、小规模与一般纳税人身份误用）而多缴或被稽查少缴。

怎么用

  1. 先按应税行为定性（销售货物 / 提供服务 / 转让不动产）查税目和税率表，避免凭客户描述就把 13% 货物销售误作 6% 服务。
  2. 判定纳税人身份（小规模季销售额 ≤ 30 万免增值税仅限普票部分 / 一般纳税人月报且必须按销项减进项计税），选择简易计税还是一般计税时测算 36 个月税负后再锁定。
  3. 对照征期日历（每月 1-15 日申报增值税与个税 / 季度终了次月预缴企业所得税 / 次年 1-5 月汇算清缴）按时申报，遇节假日按税务总局年度公告顺延。
  4. 应享尽享优惠政策（小型微利企业实际税负 5% / 高新技术企业 15% / 研发费用加计扣除 100%），同时确认当年减半政策是否延续。
  5. 涉及混合销售与兼营判定错误、印花税 2022 年新税目税率适用、个税累计预扣法税负后移等高频争议事项时，主动跟客户做政策口径对齐。

    C --> D[优惠政策匹配]

ft-tax-planner
skills/compliance/ft-tax-planner/SKILL.md

Use when: client needs tax optimization strategy, comparing tax burden

是什么

税务筹划师在企业业务发生前设计组织架构与交易方式，通过优惠政策匹配、业务拆分、时间差利用、地区差异等六大筹划方法降低税负，核心交付物是经得起稽查检验的方案（含税负测算、合法性边界、反避税条款风险提示），把合法筹划与避税 / 偷税之间的红线划清楚，避免企业被一般反避税条款（企业所得税法第 47 条）调整或刑事追诉。

怎么用

  1. 先确认业务是否已经发生，已发生的只能做合规纠正不能做筹划（事后补签合同、调整发票品目本质是虚构交易或隐匿收入）。
  2. 检验方案是否有真实商业目的（除节税之外的经济实质），无真实商业目的的方案随时可能被一般反避税条款否定。
  3. 跑全税种税负测算（增值税 + 企业所得税 + 个人所得税 + 附加税），对比方案 A 与方案 B 的税后净利润、现金流影响、合规成本、风险成本。
  4. 警惕高危场景（灵活用工平台 90% 有虚开风险 / 小规模转一般纳税人不可逆 / 税收洼地核定征收 2021 年起被大幅清理 / 关联交易达标必须准备同期资料）。
  5. 输出方案时同步给到节税金额、实施步骤、风险提示、所需资料清单，让客户在执行前能看见完整画像而不是只看见省了多少钱。

compliance-docs
skills/compliance/compliance-docs/SPEC.md

Generates compliance documents, policy mappings, and audit reports for business-tax-finance scenarios. Follows regulatory requirements and industry standards. Use when creating compliance reports, policy documents, or audit materials.

是什么

帮你把企业财税合规（Compliance）的文档活儿——审计报告、政策映射、内控评估、风险评估——从空白稿变成可直接交监管/审计师/老板看的成品；把"知道要做"变成"已经做完"。

怎么用

  1. 选好这次要生成哪一类合规文档（审计报告 / 政策映射 / 内控评估 / 风险评估）。
  2. 把公司主体、报告期、行业、关键风险点喂给它当上下文。
  3. 让它按 5 段固定结构（背景 / 范围 / 方法 / 发现 / 结论）出第一稿。
  4. 在它给出的发现里挑出真正影响审计意见的 3 条，让它重写到合规口径。
  5. 把成稿挂上你的电子签字和报告期，递交给审计师或监管口子。

架构图

flowchart LR
  A[公司主体 + 报告期] --> B[模板选择]
  B --> C[5 段结构生成]
  C --> D[发现复核]
  D --> E[合规口径定稿]
