# Checklist · 业财税合规专家 · 日 / 周 / 季度交付清单

> 端到端交付清单 · 按生命周期分段（输入 → 加工 → 输出 → 闭环）
> 每条 checkbox 必须有「证据」字段（凭据 / 截图 / 链接），无证据不算完成。

## 阶段 1 · 输入（每日开工前 30 分钟）

- [ ] **政策新闻扫描**：12366 + 财政部官网 + 1 个行业信源，至少 3 个站点（证据：浏览器历史截图 OR 链接列表）
- [ ] **客户待办筛选**：今日需触达的客户清单（证据：CRM 截图 OR 列表）
- [ ] **昨日 follow-up 闭环**：未结的 follow-up 项归零 OR 升级（证据：跟进表）

## 阶段 2 · 加工（每日核心工作时段）

- [ ] **新政策 1-pager**：若有新发布，T+1 内出 1-pager（证据：1-pager 文件 + ≥2 信源）
- [ ] **SOP delta 扫描**：1-pager 出完后扫 SOP（证据：delta 表）
- [ ] **客户合规体检**（季度内分批）：今日完成的体检客户数（证据：5 维报告）
- [ ] **询函回复**（若有）：T+3 内交底稿（证据：回复函 + 附件清单）

## 阶段 3 · 输出（每日 16:00 前）

- [ ] **客户告知函发出**：今日新政策已分群发出（证据：邮件发送回执）
- [ ] **风险提示函出具**：体检发现的红线已出函（证据：函件 + 客户回执）
- [ ] **制度文档版本号 bump**：本日修订的制度已更新版本号 + 复核日期（证据：git commit OR 文档版本表）

## 阶段 4 · 闭环（每日 17:30）

- [ ] **30 日内待办自查**：超期项标红 + 升级（证据：跟进表）
- [ ] **法律保留留痕**：客户拒绝接受风险的，书面留痕已归档（证据：归档目录 + 文件名）
- [ ] **次日开工清单准备**：明日的输入清单已写（证据：清单文件）

## 周度（每周五 16:00 前）

- [ ] **本周新政策 ≥1 个**已纳入 SOP（如本周无新政策，记录 SKIP 原因）
- [ ] **本周客户体检完成数 / 计划数**（截图）
- [ ] **本周内控议题 ≥1 个**（议题清单）

## 季度（季度末最后 5 个工作日）

- [ ] **5 维评分** 已基于客观数据填表
- [ ] **上季度行动项 follow-up** 全部 close（Done / Dropped 都需写理由）
- [ ] **Top-3 风险** 已按 ICP 排序
- [ ] **下季度 3 个 SMART 行动项** 已定义责任人 + 截止日期
- [ ] **Executive Summary ≤200 字** 已写

## 反模式自检（每日 18:00 默念）

- 我有没有 "听起来对" 但没证据的合规判断？
- 我今天有没有承诺 / 表达超出当前数据支持的结论？
- 我有没有放过 ≥1 个本该升级到合规会议的红线？

如三个问题任一回答"是"，明日开工第一件事就是补证据 / 撤回承诺 / 升级红线。
