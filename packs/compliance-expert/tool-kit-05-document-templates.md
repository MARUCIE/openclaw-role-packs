# Tool-Kit 05 · Document Templates · Compliance Expert

> 5 类合规文档模板：政策摘要 / 客户告知函 / 风控矩阵 / 询函回复 / 季度复盘报告。
> 每个模板含 ① 结构骨架 ② 必填字段 ③ 反模式自检 ④ 1 个真实示例片段（脱敏）。

---

## Template 1 · 政策摘要 1-Pager

### 结构

```markdown
# 政策摘要：<政策标题 + 文号>

| 元数据 | 值 |
|--------|---|
| 文号 | 财税[2026]X号 |
| 发布单位 | 财政部 + 国家税务总局 |
| 生效日期 | YYYY-MM-DD |
| 过渡期 | 至 YYYY-MM-DD |
| 我方复核日期 | YYYY-MM-DD |
| 下次复核日期 | YYYY-MM-DD（+12 个月） |

## 1. 适用范围
- 行业：
- 规模：
- 业务类型：

## 2. 关键变更点（3-5 条）
| # | 旧规定 | 新规定 | 差异点 |
|---|--------|--------|--------|
| 1 | | | |

## 3. 客户分群影响
| 分群 | 客户数 | 影响 | 量化（金额/% 变动） |
|------|-------|------|--------------------|

## 4. SOP 调整清单
- [ ] 节点 ID + 操作（新增/修改/删除）+ 截止日期

## 5. 来源（≥2 个独立信源）
1. 原文：<链接>
2. 第二信源：12366 答疑 / 头部行业实操 / 省局执行口径
```

### 必填字段
所有 `<...>` 占位符 + 复核日期 + ≥2 信源

### 反模式自检
- ❌ 只有原文链接没有第二信源 → 单源解读风险
- ❌ "对所有客户都有影响" → 没做分群分析
- ❌ SOP 调整清单空白 → 政策摘要没落地

## Template 2 · 客户告知函

### 结构

```markdown
尊敬的 <CLIENT_NAME> 客户：

【对你的影响】
基于政策<文号>，<量化影响：节省/增加 ¥X，按你 2025 年数据估算>。

【我们将代你做的】
1. <动作 1>（截止 YYYY-MM-DD）
2. <动作 2>

【你需要配合的】
- <资料 1>
- <资料 2>

【截止日期 + 风险提示】
请在 YYYY-MM-DD 前完成<动作>，否则将面临<具体后果：罚款金额/失去某项资格/补缴金额>。

如有疑问，回复本邮件或联系 <联系人>。

<事务所签章>
```

### 必填字段
- 客户姓名 + 政策文号 + 量化影响 + 截止日期 + 具体后果

### 反模式自检
- ❌ "可能影响" "建议关注" → 没量化、客户不重视
- ❌ "请提供相关资料" → 没列具体清单，客户不知道交什么
- ❌ "尽快处理" → 没截止日期，无落地

## Template 3 · SOX 404 风控矩阵

### 结构

```markdown
# 风控矩阵：<流程名>

## 控制目标
<1-2 句陈述，例：确保所有应收账款准确入账，无虚增>

## 风控矩阵（最少 8 行）

| 风险 ID | 风险描述 | 控制活动 | 类型 (P/D/C) | 频率 | 责任人角色 | 关键指标 | 测试方法 |
|---------|---------|----------|--------------|------|-----------|---------|---------|
| R01 | | | | | | | |
| R02 | | | | | | | |
...

## Inversion 反向 check
如果有人故意绕过所有控制，最 likely 路径是：
1. <路径 1>
2. <路径 2>

对应补丁：
- <补丁 1 → 新增控制 RXX>
```

### 必填字段
- 控制目标 + ≥8 行风险 + Inversion check + ≥2 个补丁

### 反模式自检
- ❌ 全是 D (检测型) 控制，没有 P (预防型) → 事后审计偏多
- ❌ 频率全是「季度」→ 缺少日常控制
- ❌ Inversion check 空白 → 没做反向思考

## Template 4 · 税务局询函回复函

### 结构

```markdown
<受文单位>:

收到贵局 <文号> 关于 <客户名> 的<事项>询函，现就询问事项答复如下：

## 一、事实陈述（按时间线）
1. YYYY-MM-DD：<事实 1>（凭据：附件 1，已附）
2. YYYY-MM-DD：<事实 2>（凭据：附件 2，待补）

## 二、法规依据
1. <政策文号>第 X 条第 Y 款规定：「<原文>」
2. <政策文号>第 X 条规定：「<原文>」

## 三、答复结论
基于上述事实与法规，<我方观点>。

## 四、附件清单
| # | 文件名 | 用途 |
|---|--------|-----|
| 1 | | |

## 五、保留事项
本答复仅针对询函中明确询问的事项。如贵局后续就其他事项需要补充材料，我方将<约定回复期限>内另行答复。

<事务所签章 + 经办人 + 日期>
```

### 必填字段
- 文号引用 + 时间线 + ≥2 法规依据 + 附件清单 + 保留事项

### 反模式自检
- ❌ 承诺超出询函要求 → 无端扩大责任
- ❌ 预先承认违规 → 削弱后续协商空间
- ❌ 法规引用没有原文 → 监管方可能不认可

## Template 5 · 季度合规复盘报告

### 结构

```markdown
# 合规复盘报告 Q<N> YYYY

## Executive Summary（≤200 字）

## 1. 5 维度评分（满分 5 分）
| 维度 | 本季度 | 上季度 | 趋势 |
|------|-------|-------|------|
| 政策追踪 | | | ↑/↓/→ |
| 客户体检 | | | |
| 内控有效性 | | | |
| 培训覆盖 | | | |
| 例外管理 | | | |

## 2. 上季度行动项 follow-up
| 行动项 | 状态 | 评论 |
|--------|------|------|
| | Done / In-progress / Dropped | |

## 3. 本季度新发现 Top-3 风险（ICP 排序）
| 风险 | I×P-C | 描述 | 已采取措施 |
|------|-------|------|-----------|

## 4. 下季度 3 个 SMART 行动项
| 行动 | 责任人 | 截止 | 验收指标 |
|------|-------|------|---------|
```

### 必填字段
所有 5 个章节 + 5 维度趋势箭头 + ≥3 个 SMART 行动项

### 反模式自检
- ❌ 评分全是 4-5 分 → 自我评估通胀
- ❌ "下季度继续完善 XX" → 不 SMART
- ❌ Top-3 风险全部 I×P-C 低 → 真问题没找到
