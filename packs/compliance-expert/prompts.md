## 场景 S1 · 业税务变更追踪与执行落地（Policy Tracking → Execution）

### S1-P1 · 政策变更摘要（policy → 1-pager）

**输入形状**：1 份政策原文（PDF / HTML / 文本），文件名形如 `财税[2026]XX号.pdf`
**期望输出**：1 份 1-pager 摘要含 ① 适用范围 ② 关键变更点（3-5 条）③ 生效日期 ④ 客户分群影响 ⑤ 我们需要的 SOP 调整 ⑥ 来源引用（原文 + 1 个独立信源）

```text
你是业财税合规风控专家。阅读以下政策原文，按 6 段输出 1-pager 摘要：
1) 适用范围（行业 / 规模 / 业务类型）
2) 关键变更点（3-5 条，每条标注「相比旧政策的差异」）
3) 生效日期 + 过渡期
4) 我代账客户的 3 个分群（A: 小规模 / B: 一般纳税人制造 / C: 高新技术）的影响
5) 我需要在 SOP 里修改 / 新增 / 删除的具体节点
6) 来源引用（原文链接 + 1 个独立信源：12366 答疑 / 头部行业实操 / 省局执行口径，2 选 1）
任何无法在原文找到依据的判断，标注【推断】并降低自信度。
原文：<<<PASTE>>>
```

**Worked example**：输入「财政部+税务总局公告 2026 年第 X 号 关于研发费用加计扣除比例调整」→ 输出 1-pager 含 ① 适用：科技型中小企业 ② 关键变更：加计扣除比例由 100% 提高至 120% ③ 生效：2026-01-01 ④ 客户群 C 直接受益（约 8 家）⑤ SOP 调整：T+15 内更新申报底稿模板 ⑥ 来源：政策原文 + 12366 答疑 #XYZ

### S1-P2 · SOP 同步审计（policy → SOP delta）

**输入形状**：1 份政策摘要 + 当前 SOP 文档路径
**期望输出**：SOP delta 表（新增/修改/删除节点 + 触发日期 + 责任人）

```text
基于以下政策摘要，扫描我们当前 SOP《<<<SOP_FILE>>>》是否需要更新。
按表格输出：
| 节点编号 | 操作 (新增/修改/删除) | 旧描述 | 新描述 | 触发日期 | 责任人建议 |
任何 SOP 中已经覆盖的部分，标注 SKIP 并说明理由。
政策摘要：<<<PASTE>>>
```

### S1-P3 · 客户告知函（policy → client memo）

**输入形状**：1 份政策摘要 + 客户分群标签（A/B/C）
**期望输出**：分群定制的客户告知函（≤300 字，含「对你的影响 / 我们将代你做的 / 你需要配合的 / 截止日期」4 段）

```text
针对 <<<CLIENT_GROUP>>> 群客户，基于政策摘要写一份 ≤300 字的告知函。
4 段结构：
1) 对你的影响（量化：节省 / 增加 多少税负，以你 2025 年数据估算）
2) 我们将代你做的（具体行动 + 截止日期）
3) 你需要配合的（需提供的资料清单）
4) 截止日期 + 风险提示（错过后的后果）
语言：避免税务术语堆砌，每个术语必须配 1 句白话解释。
摘要：<<<PASTE>>>
```

## 场景 S2 · 客户合规体检（Client Compliance Health Check）

### S2-P1 · 增值税链路完整性体检

**输入形状**：客户最近 12 个月的增值税申报数据 + 进销项发票汇总
**期望输出**：5 维体检报告（① 进销勾稽差异 ② 红字发票合规性 ③ 留抵退税适用性 ④ 异常波动月份 ⑤ 高风险供应商）

```text
对以下客户 12 个月增值税数据做合规体检，按 5 维输出：
① 进销勾稽：是否有 ≥3% 差异月份？列出月份 + 差异金额 + 可能原因
② 红字发票：是否每笔都有完整的「红字信息表 + 蓝字原票 + 业务签批」链？列出缺链笔数
③ 留抵退税：是否符合最新留抵退税政策（行业 / 留抵金额 / 信用等级）？给出适用性判断
④ 异常波动：找出销售/进项金额环比 >50% 的月份，标注是否有业务佐证
⑤ 高风险供应商：列出累计开票占比 >10% 的供应商及其风险信号（注销 / 失联 / 异常户）
数据：<<<PASTE>>>
```

### S2-P2 · 客户内控成熟度评估

**输入形状**：客户提供的内控制度文档 + 关键业务流程描述
**期望输出**：5 级成熟度评分（1 初始 / 2 可重复 / 3 已定义 / 4 已管理 / 5 已优化）+ 升级路径

```text
按 5 级成熟度评估客户内控水平：
- 1 初始：靠个人经验，无文档
- 2 可重复：有文档但无落地证据
- 3 已定义：有文档 + 培训 + 抽样检查
- 4 已管理：有 KPI + 月度复盘 + 例外管理
- 5 已优化：有自动化控制 + 持续改进闭环
按 6 个领域分别打分：① 收入确认 ② 采购付款 ③ 存货管理 ④ 固定资产 ⑤ 资金 ⑥ 税务申报
每项给出当前分 + 目标分 + 3 个月内可执行的 2 个升级动作
文档：<<<PASTE>>>
```

### S2-P3 · 高风险交易筛查（Anomaly screening）

**输入形状**：客户过去 12 个月的交易明细（CSV/Excel）
**期望输出**：Top-20 高风险交易列表（含交易号 + 风险标签 + 取证建议）

```text
扫描以下交易明细，按 6 类风险信号筛 Top-20：
R1 大额整数（如 99,999 / 100,000）凑数嫌疑
R2 同一供应商月内频繁拆票（疑似化整为零规避监管）
R3 业务合同与发票口径不一致
R4 跨期收入 / 跨期费用调节嫌疑
R5 关联方交易未披露
R6 异常折扣 / 异常退货
每条标注：交易号 / 金额 / 风险类型 / 建议取证动作（向客户索取什么材料）
明细：<<<PASTE>>>
```

### S2-P4 · 询函回复起草

**输入形状**：税务局询函原文 + 客户业务背景
**期望输出**：1 份回复函草稿（含事实陈述 + 法规依据 + 附件清单 + 法律保留条款）

```text
基于税务局询函原文 + 客户业务背景，起草回复函。结构：
1) 事实陈述（按时间线，每个事实标注「凭据已附 / 待补」）
2) 法规依据（每条引用必须有政策原文 + 文号 + 条款编号）
3) 附件清单（编号 + 文件名 + 用途）
4) 法律保留（针对未来可能扩大询查范围的兜底语）
重要：不承担超出询函要求的额外承诺；不预先承认违规事实。
询函：<<<PASTE>>>
业务背景：<<<PASTE>>>
```

## 场景 S3 · 内控制度建模（Internal Control Design）

### S3-P1 · SOX 404 风控矩阵起草（Risk-Control Matrix）

**输入形状**：业务流程名 + 控制目标
**期望输出**：风控矩阵表（风险 ID / 风险描述 / 控制活动 / 控制类型 / 频率 / 责任人 / 测试方法）

```text
为业务流程「<<<PROCESS_NAME>>>」（控制目标：<<<OBJECTIVE>>>）起草 SOX 404 风控矩阵。
每个风险至少配 1 个控制活动。控制类型用 P (预防) / D (检测) / C (纠正)。
表格列：风险ID | 风险描述 | 控制活动 | 类型 | 频率 | 责任人角色 | 关键指标 | 测试方法
至少 8 行。最后一行加「Inversion 反向 check」：如果有人故意绕开所有控制，最 likely 用什么路径？
```

### S3-P2 · 制度文档模板填充

**输入形状**：制度主题 + 适用范围
**期望输出**：完整的制度文档（含目的 / 范围 / 定义 / 职责 / 流程 / 控制点 / 例外处理 / 附录）

```text
按 8 段模板起草制度《<<<TITLE>>>》：
1) 目的（这条制度解决什么问题）
2) 适用范围（部门 / 业务线 / 客户群）
3) 关键术语定义
4) 职责矩阵 (RACI: Responsible / Accountable / Consulted / Informed)
5) 流程步骤（编号 + 输入 + 操作 + 输出 + 控制点）
6) 例外处理（什么情况可豁免 + 谁有权批准 + 留痕要求）
7) 监控指标（KPI + 阈值 + 上报路径）
8) 附录（模板 / 表单 / 法规引用）
反模式自检：是否每条控制点都有「证据型物证」（凭据 / 截图 / 审批流），而不仅仅是「员工应该 XXX」？
```

### S3-P3 · 制度落地培训物料

**输入形状**：1 份制度文档
**期望输出**：30 分钟培训大纲 + 10 道考核题（含答案）

```text
基于以下制度文档，输出 30 分钟培训物料：
1) 大纲（5 个模块，每模块 ≤6 分钟）
2) 关键术语速查表（≤10 项）
3) 3 个真实案例（2 个正例 + 1 个反例）
4) 10 道考核题（5 选择 / 3 简答 / 2 案例分析），含标准答案 + 评分标准
5) 培训后 30 天的「制度落地自查清单」（≥8 条）
制度：<<<PASTE>>>
```

### S3-P4 · 季度合规复盘报告

**输入形状**：本季度合规事件清单 + 上季度 follow-up 项
**期望输出**：1 份季度报告（含 5 维度评分 + 上季度回顾 + 本季度发现 + 下季度行动）

```text
起草本季度合规复盘：
1) 5 维度评分（满分 5 分）：政策追踪 / 客户体检 / 内控有效性 / 培训覆盖 / 例外管理
2) 上季度行动项 follow-up（每条标记 Done / In-progress / Dropped + 理由）
3) 本季度新发现的 3 个 top 风险（用 ICP 排序）
4) 下季度的 3 个 SMART 行动项（含责任人 / 截止日期 / 验收指标）
5) 给 CEO/CFO 的 1 段 ≤200 字 executive summary
```
