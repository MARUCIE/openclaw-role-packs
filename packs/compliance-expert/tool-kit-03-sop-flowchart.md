# Tool-Kit 03 · SOP Flowchart · Compliance Expert

> 3 张 mermaid 流程图：覆盖业财税合规专家最高频的 3 个工作流。
> 每张图标注「输入 / 关键控制点 / 输出 / 失效信号」。

## SOP-A · 政策变更 → SOP 同步落地

```mermaid
flowchart TD
    A[政策原文发布] --> B{S1-P1 生成 1-pager}
    B --> C[识别影响客户分群]
    C --> D{S1-P2 SOP delta 扫描}
    D --> E[更新 SOP 文档<br/>+ 版本号 + 复核日期]
    E --> F{触发 PDCA Patrol}
    F -->|有遗漏| G[新增/修改/删除节点]
    F -->|全覆盖| H[发布 SOP v.next]
    G --> H
    H --> I{S1-P3 客户告知函}
    I --> J[分群发送<br/>+ 7 日内回执跟进]
    J --> K{30 日后自查}
    K -->|≥1 客户漏跟进| L[ALERT 红线<br/>升级到合规会议]
    K -->|全覆盖| M[闭环归档]
```

**关键控制点**：
- 节点 E：SOP 文档必须有「政策文号 + 生效日期 + 复核日期」3 字段，缺一不允许保存
- 节点 J：客户回执需附件证明（签字 PDF / 电子签 / 邮件回执），口头确认不算
- 节点 K：30 日是硬阈值（防止 follow-up 被无限延期）

**失效信号**（任一即升级到合规会议）：
1. 政策原文发布 ≥7 日仍无 1-pager
2. SOP 文档复核日期超过 12 个月
3. 客户回执缺失率 ≥10%

## SOP-B · 客户合规体检（季度执行）

```mermaid
flowchart LR
    A[季度启动<br/>客户分群 A/B/C] --> B{S2-P1 增值税链路体检}
    B --> C[5 维体检报告]
    C --> D{发现红线？}
    D -->|是| E[S2-P3 高风险交易 Top-20 筛查]
    D -->|否| F{S2-P2 内控成熟度评估}
    E --> G[向客户出具风险提示函]
    F --> H[成熟度评分 + 升级路径]
    G --> I[与客户对账会议]
    H --> I
    I --> J{客户接受？}
    J -->|是| K[纳入下季度跟进]
    J -->|否| L[书面留痕<br/>+ 法律保留]
    K --> M[归档]
    L --> M
```

**关键控制点**：
- 节点 D：「红线」定义必须是制度白名单（如「12 个月内有 ≥2 次申报偏差」），不允许审计员主观判断
- 节点 I：对账会议必须有录音/纪要，客户签字确认风险知情
- 节点 L：客户拒绝接受时，书面留痕保护事务所免责

**失效信号**：
1. 客户体检完成率 <90%/季度
2. 红线发现后 ≥14 日未出具风险提示函
3. 法律保留留痕缺失 ≥1 笔

## SOP-C · 内控制度从起草到落地

```mermaid
flowchart TD
    A[业务部门提需求] --> B{S3-P1 起草风控矩阵}
    B --> C[Inversion 反向 check<br/>5 个失效场景]
    C --> D{S3-P2 制度文档模板填充}
    D --> E[内部 review<br/>合规 + 业务 + IT 三方]
    E --> F{通过？}
    F -->|否| G[修改 → 回到节点 D]
    F -->|是| H[S3-P3 培训物料]
    G --> D
    H --> I[培训交付<br/>+ 考核 ≥80% 通过率]
    I --> J{S3-P4 季度复盘}
    J --> K[更新制度 OR 维持]
    K --> A
```

**关键控制点**：
- 节点 C：Inversion check 必须输出 5 个失效场景 + 5 个对应控制点；少于 5 个不允许进入起草阶段
- 节点 E：三方 review 必须留邮件/会议纪要；缺任一方签字不允许发布
- 节点 I：考核通过率 <80% 必须重新培训，不允许「凑数」勾选

**失效信号**：
1. 起草到发布 ≥45 日（流程僵化信号）
2. 培训覆盖率 <85% 部门人员
3. 季度复盘缺席 ≥1 次

## SOP 之间的协同

```mermaid
graph LR
    A[SOP-A 政策同步] --> B[SOP-B 客户体检]
    A --> C[SOP-C 内控制度]
    B --> D[发现共性问题]
    C --> D
    D --> A
```

3 个 SOP 形成闭环：政策变更触发体检、体检发现共性问题倒逼内控修订、内控修订反向影响 SOP-A 的复核频率。
