# Checklist · Research Analyst · 日 / 周 / 月度交付清单

## 阶段 1 · 输入（每日 30 分钟）

- [ ] **政策/行业 RSS 扫描**：≥3 个独立信源（证据：浏览器历史 OR 链接列表）
- [ ] **client research backlog**：今日需推进的研究 brief（证据：清单）
- [ ] **昨日 unanswered follow-up**：上周/上月的 unanswered 是否有新答案

## 阶段 2 · 加工

- [ ] **三角验证强制**：每份新报告至少 1 个 adversarial 信源（证据：报告中的 §3）
- [ ] **MECE 检查**：每个 question tree ≤7 leaf 且互斥（证据：question tree 文档）
- [ ] **Falsification condition**：每个 GO 决策含 1 个 falsification 条件（证据：决策段）

## 阶段 3 · 输出

- [ ] **报告交付**：今日完成的报告含 5 段全部（证据：报告文件）
- [ ] **Unanswered 段强制**：每份报告 Top 3 unanswered（证据：报告末段）
- [ ] **复核日期标注**：每份报告下次复核日期 ≤90 天（证据：metadata 表）

## 阶段 4 · 闭环

- [ ] **跨 pack 协同回传**：发给 compliance / data / strategy 的输入清单更新（证据：清单）
- [ ] **下一轮研究启动点**：本日 unanswered 已转为下一轮 brief（证据：brief 文件）
- [ ] **知识库 commit**：本日 atomic claims 已归档（证据：git commit）

## 周度

- [ ] 本周新政策/趋势 ≥1 项已纳入研究 pipeline
- [ ] 本周 SOTA paper scan ≥3 篇
- [ ] 本周 unanswered follow-up 闭环数

## 月度

- [ ] **月度政策追踪报告** 已发出
- [ ] **上月 Top 3 unanswered** 状态
- [ ] **本月新 Top 3 unanswered** 已识别
- [ ] **复核到期报告** 已全部 re-validate（含 stale 判断）

## 反模式自检（每日 18:00）

- 我今天有没有 adversarial 视角缺失的报告？
- 我今天有没有写"较大潜力"这种不可证伪的废话？
- 我今天有没有跳过 unanswered 段？
- 我今天有没有引用 >24 个月的数据当"当前现状"？
- 我今天有没有用"我认为"代替"信源 A 说 / 信源 B 说 / 我综合判断"？

如任一回答"是"，明日开工第一件事就是补 adversarial 信源 / 重写 claim / 补 unanswered 段 / 重新引用 / 重写综合段。
