# W2 · 产品研发岗（后端/前端/需求） — Before/After 基线对比表

> 第 2 波（Session 2 · Calendar W2 · 2026-05-28 workshop）的实证锚点。
> 维护人：the program owner（AI 专家）· cohort 在 T-3（2026-05-25 周一）收到。

## How to read

- **基线 (Before)**：cohort 提交的真实自报数据（2026-05-22 ~ 2026-05-25 收集）
- **AI 辅助 (After)**：本次 workshop 设定的可达目标（保守值，cohort 95% 分位能达到）
- **时间节省 %** = (基线 − After) / 基线 × 100，向下取整
- **质量信号** = 缺陷率 / 接口对齐率 / 还原度 / 联调通过率 中至少一个
- After 实测偏离 ≥ 30% 时 T+8 中期回顾时降级而非删除

## W1 → W2 衔接观察点

W2 cohort 接到 [wave-digest.md](../../doc/wave-digest.md) W1 段时（2026-05-22 周五），需重点观察 W1 的 3 个反馈是否会影响 W2 baseline：

- 若 W1 cohort 反馈"PRD 评审一次通过率 32% → 60% 实际只到 45%"，则 W2 的"评审一次通过率 40% → 70%"（行 5）目标值需保守化
- 若 W1 cohort 反馈"AI 生成的 Mermaid 流程图有效"，则 W2 SOP 流程图节点中"AI 出图"步骤的耗时预算可压缩
- 若 W1 cohort 反馈"Stitch 输入版的 prompt 形式好用"，则 W2 族 3（前端样式还原）的"Figma → AI prompt"接口可参考该形式
