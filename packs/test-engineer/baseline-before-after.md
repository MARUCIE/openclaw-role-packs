## 按族负载

| 族 | 行 | 基线（h） | 目标（h） | 节省 |
|----|----:|--------:|--------:|----:|
| F1 · 覆盖与单元测试 | 1-4 | 7.5 | 2.15 | 71% |
| F2 · 回归与变更影响 | 5-7 | 6.0 | 1.75 | 71% |
| F3 · 合规与审计链路测试 | 8-10 | 8.5 | 2.9 | 66% |

F3 绝对基线最高 — 税规边界多、审计链路期望严。F1 + F2 占常规工作量大头；AI 在枚举 + 重复脚手架上节省最强。

## 诚实的前提（用这些目标前要看）

1. 目标值假设 cohort 已装 CLAUDE.md（`tool-kit-01`）并启用 `semantic_skill_router`（用裸 Claude Code 的 cohort 只能省 40-50%，不是 67-75%）
2. 数字假设 cohort 对平台税规引擎已有 1 个月领域熟悉度 — 新规则区前 2 周节省掉到 ~30%，因为还在学行话
3. F3 第 8 行（税规合规测试）要求能访问规则注册表 + 历史测试用例 — 没访问权限的 cohort 节省 ~40%，不是 62%
4. F2 第 7 行（变更影响）假设平台已有依赖图工具 — 隔离环境装不上
5. F3 第 9 行（审计链路断言）假设凭证 / 账务记录 ID 跨环境稳定；dev / staging ID 会重置时节省显著下降

## "后" 对 cohort 来说长什么样

T+7 时 cohort 成员应该能诚实回答 "是" 在以下每一项：

- 我的测试仓库里有 CLAUDE.md，`semantic_skill_router` 不用我打 skill 名就能激活
- 每个族至少有 1 个 prompt 本周在真实平台工作上用过
- 我已经为上面 10 行中的至少 5 行填了 `baseline-actual.csv` 的自测时长
- 我能说出 1 个试过、放弃了的 prompt 和放弃原因（放弃清单比保留清单更重要）
- 我已经用 F3 prompt 写了至少 1 套税规测试用例，并在合成 fixture 上验证审计链路断言通过

如果完课一周后这里有任何一项是 "否"，工具包对该 cohort 成员失败了 — 在 T+7 反馈 Q4 记录，并分流到 Buffer 2（日历 W10）工具包迭代。
